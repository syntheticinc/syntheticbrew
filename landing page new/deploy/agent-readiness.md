# Agent readiness — syntheticbrew.ai

What this site publishes for AI agents, what it deliberately does not, and how to verify both. Reference scanner: [isitagentready.com](https://isitagentready.com) (Cloudflare).

## Implemented

| Surface | Artifact | Where |
|---|---|---|
| Auth.md agent authentication guide | `/auth.md` (H1 `# auth.md`, self-contained JWT flow) | `public/auth.md` |
| Agent Skills discovery index | `/.well-known/agent-skills/index.json` + per-skill `SKILL.md` | `public/.well-known/agent-skills/*/SKILL.md`; index generated by `scripts/build-skills-index.mjs` (digests from dist bytes) |
| Markdown content negotiation | `Accept: text/markdown` on any page returns its markdown mirror, `Content-Type: text/markdown`, `Vary: Accept`; HTML stays the default | mirrors from `scripts/build-markdown-mirrors.mjs`; `@markdown` stanza in `deploy/Caddyfile` and `deploy/Caddyfile.preview` |
| WebMCP browser tools | `navigator.modelContext.provideContext()` with 3 read-only tools on every page | inline script in `src/layouts/SiteLayout.astro` |
| DNS-AID discovery records | `_mcp._agents` / `_index._agents` SVCB records + DNSSEC | Cloudflare DNS (manual) — runbook: [`dns-aid.md`](./dns-aid.md) |
| LLM link index | `/llms.txt` | `public/llms.txt` |
| API catalog (RFC 9727) | `/.well-known/api-catalog` | `public/.well-known/api-catalog` |
| MCP server card | `/.well-known/mcp/server-card.json` | `public/.well-known/mcp/server-card.json` |

`scripts/audit-build.mjs` (part of `npm run build`) enforces all in-repo artifacts, digest integrity, and prod/preview Caddyfile parity.

## Intentionally not implemented

Do not "fix" these to satisfy a scanner — they would advertise capabilities that do not exist and mislead agents.

- **OAuth/OIDC discovery** (`/.well-known/openid-configuration`, `/.well-known/oauth-authorization-server`) and **OAuth Protected Resource Metadata** (`/.well-known/oauth-protected-resource`): SyntheticBrew operates no public OAuth/OIDC authorization server. The complete supported flow is JWT Bearer, documented in `/auth.md` — the Auth.md spec explicitly permits this self-contained no-OAuth form. Revisit only if a real OAuth server ships.
- **x402 / MPP / UCP / ACP** (agent-native payments and commerce): the site has no machine-payable API and no agent checkout — subscriptions are sold through the dashboard. Stub discovery documents would point agents at endpoints that cannot take their money.
- **`x-markdown-tokens` response header**: optional in the scanner; static Caddy hosting cannot compute per-file token counts.

## Verification

Local, after `npm run build` (requires the preview stack — `astro dev` does not generate mirrors or the skills index):

```bash
docker compose -f deploy/docker-compose.preview.yml up -d --force-recreate

# Markdown negotiation
curl -si http://localhost:4330/pricing/ -H 'Accept: text/markdown' | head -20   # 200, text/markdown, Vary: Accept
curl -sI http://localhost:4330/pricing/                                          # text/html (default unchanged)
curl -si http://localhost:4330/ -H 'Accept: text/markdown' | head -5             # root mirror
curl -sI http://localhost:4330/nope/ -H 'Accept: text/markdown'                  # 404

# Discovery assets
curl -s  http://localhost:4330/auth.md | head -1                                 # "# auth.md"
curl -si http://localhost:4330/.well-known/agent-skills/index.json | head        # application/json
curl -sI http://localhost:4330/.well-known/agent-skills/syntheticbrew-docs-mcp/SKILL.md  # text/markdown
sha256sum dist/.well-known/agent-skills/*/SKILL.md                               # must match index.json digests

# WebMCP
curl -s http://localhost:4330/ | grep -c 'navigator.modelContext'                # >= 1
```

Production, after deploy (and DNS changes per [`dns-aid.md`](./dns-aid.md)): repeat the curl suite against `https://syntheticbrew.ai`, then re-scan:

```bash
curl -s -X POST https://isitagentready.com/api/scan \
  -H 'Content-Type: application/json' \
  -d '{"url":"https://syntheticbrew.ai"}'
```

Expected: passes for auth-md, agent-skills, markdown negotiation, WebMCP, and DNS-AID (after DNS/DNSSEC propagation); the OAuth and payment checks fail by design (see above).
