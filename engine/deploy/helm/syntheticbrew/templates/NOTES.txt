Optional: declarative config-as-code (brewctl)

3. Single-shot bootstrap (recommended):
   Pre-mint an admin API token, store in Secret, enable BOTH bootstrap and configApply:

   a. Mint token: TOKEN="bb_$(openssl rand -hex 32)"
   b. Create Secret with token + LLM keys:
        kubectl create secret generic syntheticbrew-config \
          --from-literal=token=$TOKEN \
          --from-literal=DATABASE_URL=postgres://... \
          --from-literal=OPENROUTER_API_KEY=sk-or-... \
          -n {{ .Release.Namespace }}
   c. Helm install with both enabled:
        helm install syntheticbrew oci://ghcr.io/syntheticinc/charts/syntheticbrew-engine \
          --version 0.4.1 \
          --set bootstrapAdminToken.enabled=true \
          --set bootstrapAdminToken.existingSecret=syntheticbrew-config \
          --set configApply.enabled=true \
          --set configApply.tokenSecret=syntheticbrew-config \
          --set configApply.apiKeysSecret=syntheticbrew-config \
          --set-file configApply.config=./syntheticbrew.yaml

   On first boot engine seeds the token into api_tokens (idempotent: re-deploy
   won't duplicate). brewctl Job then uses the same token to apply config.

   IMPORTANT: syntheticbrew.yaml must reference secrets via env-var placeholders
   (api_key: ${OPENROUTER_API_KEY}), never plaintext. Plaintext keys leak
   into ConfigMaps which any user with `secrets:get` RBAC can read.

Alternative: post-deploy manual token gen (legacy / recovery):

   1. Generate an admin API token via the Admin Dashboard
        (https://<host>/admin -> API Keys -> Create).

   2. Create the token Secret:
        kubectl create secret generic syntheticbrew-token \
          --from-literal=token=bb_xxx \
          -n {{ .Release.Namespace }}

   3. (Optional) Create a Secret with LLM API keys:
        kubectl create secret generic syntheticbrew-api-keys \
          --from-literal=OPENROUTER_API_KEY=sk-or-... \
          --from-literal=TAVILY_API_KEY=tvly-... \
          -n {{ .Release.Namespace }}

   4. Re-deploy with configApply enabled:
        helm upgrade {{ .Release.Name }} . \
          --set configApply.enabled=true \
          --set configApply.tokenSecret=syntheticbrew-token \
          --set configApply.apiKeysSecret=syntheticbrew-api-keys \
          --set-file configApply.config=./syntheticbrew.yaml
